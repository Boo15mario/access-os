# Access OS Docs

This directory is the working documentation hub for the Access OS project.

## Start Here

- [Stack Overview](./stack-overview.md) — what each repo does and how they fit together
- [Build and Release Flow](./build-and-release-flow.md) — expected order of operations from packages to ISO
- [Current Status](./current-status.md) — where the project stands right now
- [Next Steps](./next-steps.md) — prioritized roadmap
- [Work Log](./work-log.md) — short running notes as work happens

## Documentation Rules

- Keep docs practical and current.
- Prefer short sections and checklists over walls of text.
- When behavior changes, update the relevant doc in the same change when possible.
- If something is not decided yet, mark it clearly as **TBD** rather than implying it is settled.

## Suggested Workflow

1. Update `work-log.md` when something meaningful changes.
2. Move stable conclusions into the more permanent docs.
3. Keep `current-status.md` honest.
4. Keep `next-steps.md` prioritized so it stays useful.
